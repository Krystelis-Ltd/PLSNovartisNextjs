## 2024-05-18 - Optimize Extraction Data Reduce O(n^2) Bottleneck
**Learning:** In `src/app/page.tsx`, `currentFetchedAnswers` was recalculating on every render by filtering the feed and running `.reduce()` using the object spread operator (`{ ...acc, [key]: value }`). React arrays mapped to components trigger this heavily. Using object spread in a reducer creates a new object on every iteration, leading to O(n^2) performance which degrades as the extraction feed grows large.
**Action:** Always prefer direct object mutation (`acc[key] = value; return acc;`) within a `.reduce()` loop when initializing a new object, achieving O(n) performance. Additionally, wrap derived expensive structures in `useMemo` to prevent re-evaluation on unrelated state changes (like user typing in Chatbot), and wrap passed callback handlers in `useCallback` to prevent cascading component re-renders.
