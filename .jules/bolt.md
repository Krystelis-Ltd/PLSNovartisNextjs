## 2026-05-04 - React Compiler & Memoization
**Learning:** In Next.js 16 / React 19 environments using the React Compiler, traditional manual memoization (e.g. `useMemo`, `useCallback`) must be carefully applied. Calling impure functions like `Date.now()` directly in the render body or in non-memoized async handlers will trigger `react-hooks/purity` errors. Furthermore, wrapping everything blindly in `useCallback` can trigger `react-hooks/exhaustive-deps` for unrelated functions (like `useToast`).
**Action:** Ensure any manual memoization properly resolves dependencies, and instead of wrapping async event handlers just to hide impure function calls, accept that async event handlers don't strictly execute during the render phase and shouldn't trigger the purity check unless incorrectly invoked. If `Date.now()` must be used, ensuring it isn't directly exposed to the render body's synchronous execution flow is key.
