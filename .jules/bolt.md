## 2026-05-18 - Expensive O(n^2) Reduce in React Renders
**Learning:** React components (like `page.tsx`) performing data transformation (e.g. mapping pipeline extraction feeds into dictionaries) using array `.reduce()` with the object spread operator (`{ ...acc }`) create a severe O(n^2) performance bottleneck. This occurs because the spread operator creates a shallow copy of the entire accumulator object on every iteration, growing quadratically with the array size. When this logic is not wrapped in `useMemo`, it blocks the main thread on every render.
**Action:** Always refactor such `.reduce()` operations to mutate the accumulator object directly (`acc[key] = value`) yielding O(n) performance. Crucially, wrap these derived state calculations in `useMemo` to prevent recalculation during unrelated re-renders.
