## 2024-05-04 - Fix IP Spoofing and Log Injection in Audit Logger
**Vulnerability:** The audit logger previously extracted client IPs using `request.headers.get('x-forwarded-for') || request.headers.get('x-real-ip')`. This was susceptible to IP spoofing because attackers could simply set these headers, and no sanitization was performed before logging, opening the possibility for log injection vulnerabilities by introducing newlines.
**Learning:** `X-Forwarded-For` is a comma-separated list where the leftmost IP is the original client (easily spoofed) and the rightmost IPs are added by trusted proxies. In environments like Azure App Service, `X-Azure-ClientIP` is a more reliable and secure header, while relying blindly on `x-real-ip` is insecure.
**Prevention:** Always implement a dedicated `getSecureIp` utility that sanitizes IPs (stripping `\r\n` to prevent log injection), prioritizes environment-specific secure headers like `X-Azure-ClientIP`, and when forced to use `X-Forwarded-For`, extracts the *last* IP in the list instead of the first or the raw string.
