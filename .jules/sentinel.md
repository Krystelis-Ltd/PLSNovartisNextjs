## 2024-05-18 - Fix IP Spoofing and ID Truncation Vulnerabilities
**Vulnerability:** The application was vulnerable to IP spoofing because it incorrectly read the first `X-Forwarded-For` entry or simply used it without validation. Additionally, there was a risk of ID collision because `uuid` values used for session and correlation IDs were truncated to 8 characters.
**Learning:** Extracting client IPs from `X-Forwarded-For` requires care, as the list is typically appended to by proxies. In Azure, `X-Azure-ClientIP` is a verified header, while for `X-Forwarded-For`, the last entry should be used. Truncating UUIDs drastically reduces their uniqueness and defeats their purpose.
**Prevention:** Always prioritize trusted headers like `X-Azure-ClientIP` in Azure App Service environments. For `X-Forwarded-For`, extract the last IP in the comma-separated list. Never truncate UUIDs generated by `crypto.randomUUID()` or fallback pseudo-random generators, as it compromises collision resistance.
