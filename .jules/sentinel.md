## 2024-05-06 - Secure IP Extraction and ID Generation
**Vulnerability:** The audit logger was using `Math.random()` to generate pseudo-random identifiers, making them predictable and vulnerable to guessing attacks. Additionally, the IP extraction logic relied on the first entry in `X-Forwarded-For` and prioritized it over `X-Real-IP`, which allowed attackers to spoof their IP address by injecting their own `X-Forwarded-For` header.
**Learning:** In a security context, IDs used for correlation, session tracking, or any other identification purpose must be generated using cryptographically secure pseudorandom number generators (CSPRNGs). For Azure App Service environments, prioritizing `X-Azure-ClientIP` is crucial, and when using `X-Forwarded-For`, the *last* IP in the list should be used, as it is the one appended by the proxy and cannot be spoofed by the client.
**Prevention:** Always use `crypto.randomUUID()` or `crypto.randomBytes` for identifier generation. Implement secure IP extraction logic that correctly prioritizes verified headers and selects the correct proxy-appended IP from lists.
