## 2024-05-15 - Secure Client IP Extraction in Azure & UUID Truncation Risks
**Vulnerability:**
1. The app extracted client IP using `request.headers.get('x-forwarded-for') || request.headers.get('x-real-ip')`, which is vulnerable to spoofing, especially since `x-forwarded-for` can contain a comma-separated list of IPs.
2. Fallback session and correlation IDs were generated by truncating `crypto.randomUUID()` to 8 characters (`uuid.substring(0, 8)`). Since random UUIDs aren't entirely random but are structured, truncating them drastically reduces entropy and introduces collision risks.

**Learning:**
In Azure App Service environments, the most reliable and secure header for client IP is `X-Azure-ClientIP`. If relying on `X-Forwarded-For`, the *last* IP in the comma-separated list should be used, as it is appended by the proxy and is the most trustworthy. The first IP or `X-Real-IP` can be easily spoofed by the client. Furthermore, truncating standard UUIDs defeats their purpose and compromises uniqueness in security contexts like audit logging.

**Prevention:**
Prioritize `X-Azure-ClientIP` when deployed in Azure. If using `X-Forwarded-For`, parse the list and extract the *last* entry. Never truncate `crypto.randomUUID()` values when using them as unique identifiers for sessions or correlation IDs.
